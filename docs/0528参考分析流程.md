# 10W VCF 实际分析流程（v0.10.4）

> 生成日期: 2026-05-28
> 适用版本: gpa-genomic-phenotype v0.10.4

---

## 输入

- **10W 量级 VCF**（`.vcf.gz`）
- **患者表型描述**（如"远端肌病，进行性肌无力，肌萎缩，肌源性损害"）

---

## Step 0 — 输入确认

- 分析目的（疾病遗传风险 / 肿瘤驱动 / 药物基因组学）
- 样本身份（患者本人 / 健康筛查）
- 关注组织（神经系统 / 血液 / 心血管 / 肝脏 / 肾脏等）

---

## Step 1 — VCF 预处理（本地，~1-2 分钟）

**工具**: `gpa_vcf_annotator._parse_vcf()` + `_prefilter()`

- 读取 `.vcf.gz`，提取 `CHROM/POS/REF/ALT/QUAL/DP/GT/GQ/VAF`
- 预过滤剔除 `QUAL < 20` 或 `DP < 10`
- **10W → ~8W** 高质量变异

---

## Step 2 — VEP REST API 注释（~5-15 分钟，最大瓶颈）

**工具**: `gpa_vcf_annotator._annotate_vep_api()`

- 每批 200 个变异，并发 5 个请求
- 8W ÷ 200 = **400 批**
- 支持 **shard resume**（每 1000 变异一个 shard，中断可续传）
- 输出：Consequence / HGVSp / HGVSc / domains / 全部转录本

---

## Step 3 — 转录本选择（~30 秒-2 分钟）

**工具**: `gpa_transcript_selector`

- 如有 disease description，触发 disease-aware 选择
- 四层评分：tissue_expression > consequence > canonical/MANE > exon location

---

## Step 4 — Phase 1: 本地 Triage（~30 秒，不查任何 API）

两条路径：

| 路径 | 做什么 | 结果 |
|------|--------|------|
| **默认** (`gpa_pipeline.py`) | 全量 `classify_variant_tier()` | 所有变异分 Tier 1/2/3 |
| **`--two-phase`** (`gpa_two_phase.py`) | `_is_potentially_pathogenic()` 过滤 | 仅 candidates 保留，其余 Tier 3 |

**过滤规则**（`--two-phase`）：
- ✅ HIGH impact（stop_gained, frameshift, splice）— 除非 gnomAD AF > 0.95 纯合
- ✅ MODERATE + 已知疾病基因 — 除非 gnomAD AF > 5%
- ✅ MODERATE + ClinVar Pathogenic
- ✅ MODERATE + 极罕见（gnomAD AF < 0.001）或未知
- ✅ LOW/MODIFIER + splice-relevant consequence
- ❌ 同义、纯内含子、UTR、upstream/downstream、MODIFIER

**典型 10W VCF**：过滤后 **500-2000 candidates**

---

## Step 5 — Phase 2: API Enrichment（~3-60 分钟，取决于候选数量）

### 进入 Phase 2 的变异

- **默认路径**：Tier 2 全部 + Tier 3 中符合条件的（frameshift/stop_gained/splice/missense in disease gene 等）
- **`--two-phase`**：Phase 1 过滤出的全部 candidates

### 超限提醒机制（v0.10.4）

当候选数量超过 `max_candidates=150`（`--two-phase`）或 `max_total=150`（默认路径）时：

```
============================================================
[GPA WARNING] Candidate count (1800) exceeds threshold=150
              All 1800 candidates will proceed to API enrichment
              (no truncation). This may take longer due to
              gnomAD rate limits.
============================================================
```

- **不截断**，继续全量查询
- 由用户根据警告决定是否中断、调整参数重跑

### 查询内容

| API | 查什么 | 数量级 | 耗时 |
|-----|--------|--------|------|
| **基因级** Ensembl/UniProt/gnomAD_constraint | 候选基因 | ~100-300 基因 | **0-30s**（952 archive 基因基本全命中） |
| **MyVariant.info** | gnomAD + ClinVar + CADD | 全部候选 | **~1-2 min / 1000 候选** |
| **gnomAD GraphQL** fallback | MyVariant 未命中的 | 剩余部分 | **2 req/s**，是最大瓶颈 |
| **SpliceAI** | splice-relevant 变异 | ~10-30% 候选 | **~10-30s** |
| **Phenotype LLM** | 候选基因-表型匹配 | ~50-100 基因 | **~30-60s** |

**gnomAD rate limit 是硬瓶颈**：
- 150 候选 → ~1-2 分钟
- 1000 候选 → ~8-10 分钟
- 5000 候选 → ~40-50 分钟（会打印警告）

---

## Step 6 — Phase 3: 重新分类（~30 秒）

**工具**: `gpa_tier_classifier.classify_variant_tier()`

- gnomAD EAS AF > 1% → 降级 Tier 3
- ClinVar Pathogenic + 高星级 → Tier 1
- SpliceAI delta=0 → splice 变异可能降级

---

## Step 7 — Multi-hit + X-linked 调整（~10 秒）

- 同一基因 ≥2 个变异 → multi-hit 标记
- X-linked 女性杂合 → 调整 tier

---

## Step 8 — 报告生成（~10 秒）

**工具**: `gpa_report`

输出：
1. **Markdown 报告**：摘要、Tier 1/2/3 详情、multi-hit、QC、表型关联
2. **JSON**：`tier1/2/3_variants`、`summary`、`multi_hit_details`

---

## 总耗时估算

| 场景 | 候选数 | 预估时间 |
|------|--------|----------|
| 理想（候选 <150, archive 全命中） | <150 | **~8-12 分钟** |
| 一般（候选 500-1000） | 500-1000 | **~15-30 分钟** |
| 大量候选（gnomAD 瓶颈） | >1000 | **30-60 分钟+**（打印警告） |
| 离线模式 | — | **~3-5 分钟**（质量差） |

---

## 关键决策点

**候选数 > 150 时**：
- 日志打印醒目标题 `=== [GPA WARNING] ===`
- Pipeline **继续运行**，不丢弃任何候选
- 用户可根据警告和预估时间，决定是否中断、收紧筛选规则后重跑
